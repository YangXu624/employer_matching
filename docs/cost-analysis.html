Cost analysis

LLM scoring cost analysis

The purpose of this LLM pipeline is to ingest raw, unstructured job descriptions (JDs) and output structured competency scoring (evaluating 6 POG competencies on a 1–5 scale). This document breaks down the token counts, pricing per model, and total projected cost at various scales. All pricing is sourced from official API documentation and verified as of June 2026.

How the prompt is built

The prompt sent to the LLM has a fixed prefix and a variable JD text:

ComponentContentApprox tokens
Rubrics and instructionRole prompt ("expert HR analyst"), scale definitions (1–5)~120
Competency definitions6 POG competency descriptions, one per POG~180
JSON schemaRequired output format with 6 competency objects~100
Subtotal (fixed)~400
JD textEmployer's job description (varies)~670
Total input~1,070

The output is a JSON object with 6 entries, each containing a level, confidence score, and short explanation:

ComponentContentApprox tokens
JSON structureFixed wrapper with 6 competency objects~60
Reasoning text"Short explanation" per competency, ~15–30 words each~140
Total output~200

Assumes a ~500-word JD. A short JD (~200 words) might be ~500 input tokens total. A long JD (~1,500 words) might be ~2,400 input tokens. Output stays relatively constant at ~200 tokens regardless of JD length, since the reasoning stays brief.

What about running locally?

Cost$0.82 per 1,000 JDs$0 (but hardware cost up front)

SetupAPI key, one importDownload ~4–8 GB model, manage GPU/CPU memory
MaintenanceNoneModel updates, resource monitoring, environment config
DeterminismLow (temperature-dependent)Low (same issue)

Recommendation

Use paid API calls (Gemini 2.5 Flash or DeepSeek V4 Flash)

Stick with paid API calls for LLM scoring, just like how we generate the student POG passports. The cost is negligible at this scale — running a thousand JDs costs between $0.53 (best case) and $1.47 (worst case) on Gemini 2.5 Flash, and only $0.11 to $0.42 on DeepSeek V4 Flash. This keeps the implementation simple, the speed fast, and avoids the complexity of running and maintaining local models. The deterministic embedding pipeline remains the default; the LLM API is only called when the user explicitly clicks "LLM Check," so costs are entirely optional and usage-controlled.
